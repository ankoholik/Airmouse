# Протокол экспериментов и анализ результатов (AirMouse)

## Аннотация

Документ описывает, как выполняются серии экспериментов обучения модели (разные seeds и гиперпараметры), какие артефакты сохраняются, и как интерпретировать полученные метрики. Цель — получить статистически устойчивую оценку качества и выполнить требования по количеству экспериментов/суммарному времени обучения.

## 1. Мотивация серии экспериментов

Один запуск обучения не даёт устойчивой оценки качества из‑за:

- стохастичности оптимизации (инициализация, порядок батчей)
- возможных локальных минимумов
- чувствительности к гиперпараметрам (например, learning rate)

Поэтому проводится серия независимых прогонов с регистрацией метрик.

## 2. Инструмент проведения экспериментов

Скрипт: `scripts/run_experiments.py`.

Пример (30 запусков):

```bash
python scripts/run_experiments.py --runs 30 --epochs 80 --lrs 0.001 0.0005 0.0002
```

Скрипт:

- запускает обучение `runs` раз
- варьирует `seed` и learning rate (циклически по списку `--lrs`)
- сохраняет артефакты каждого запуска в `logs/experiments/run_*/`
- пишет сводный CSV с метриками и временем

## 4. Анализ результатов серии

### 4.1. Метрика качества

Основная агрегируемая метрика: `test_accuracy`.

В сводном CSV зафиксированы результаты 30 запусков; по ним корректно вычислять:

- среднее и стандартное отклонение точности
- минимум/максимум точности
- зависимость качества от `lr`

### 4.2. Ошибки классификации

Confusion matrix полезна для понимания, какие классы чаще путаются.

Пример (run_030):

![Матрица ошибок (пример run_030)](../logs/experiments/run_030/confusion_matrix.png)

## 5. Требования к количеству экспериментов и времени

В зависимости от длительности обучения:

- если один запуск < 5 минут — требуется **> 30 экспериментов** (либо обоснование, почему дальнейшие эксперименты нецелесообразны)
- если один запуск > 5 минут — требуется столько экспериментов, чтобы суммарно превысить **30 часов**

В данном проекте серия запусков фиксируется автоматически в CSV через поле `wall_time_s`.
