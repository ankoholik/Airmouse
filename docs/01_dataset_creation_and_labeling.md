# Формирование и разметка датасета жестов кисти (AirMouse)

## Аннотация

Документ описывает процесс формирования датасета жестов кисти для задачи многоклассовой классификации (управление курсором/кликами/скроллом). Приведены определения объекта наблюдения, признакового описания, целевого признака, а также требования к структуре хранения и правила заполнения датасета.

## 1. Постановка задачи

Рассматривается задача классификации жестов кисти по 3D-координатам опорных точек MediaPipe Hands.

- **Тип задачи**: supervised learning, многоклассовая классификация.
- **Выход модели**: класс жеста (дискретная метка).

## 2. Определения: объект, признаки, целевой признак

- **Объект (observation)**: один кадр/снимок позы кисти, сохранённый в отдельном файле `.npy`.
- **Признаки (features)**: вектор длины 63 из чисел с плавающей точкой, полученный из \(21\) landmark \(\times\) \(3\) координаты \((x,y,z)\).
- **Целевой признак (target)**: метка класса `class_id` (на диске хранится как `1..K`, в обучении приводится к `0..K-1`).

## 3. Структура датасета и требования к хранению

Корень датасета: `data/`.

### 3.1. Рекомендуемая структура (структурированный датасет)

Рекомендуемый формат исключает накопление большого числа файлов в одном каталоге и обеспечивает воспроизводимые разбиения:

```text
data/
  raw/
    <class_id>/
      <shard>/        # 0000, 0001, ... (до 1000 файлов в каталоге)
        <n>.npy
  index.csv           # path,class_id (пути относительно data/)
  splits/
    train.csv         # path,class_id
    test.csv          # path,class_id
```

## 4. Как формируются признаки

Вектор признаков получается из детектора MediaPipe Hands:

- извлекаются 21 landmark кисти
- выполняется нормализация относительно landmark 0 (wrist)
- значения разворачиваются (flatten) в вектор из 63 чисел

Формат одного файла `.npy`:

- **тип**: массив float
- **форма**: `(63,)`

## 5. Разметка и правила заполнения датасета

### 5.1. Что такое разметка

Разметка соответствует выбору класса `class_id` перед сохранением примера. Это **жёсткая (hard) разметка**: каждому объекту соответствует ровно один класс.

### 5.2. Как правильно заполнять датасет

Рекомендуется обеспечивать внутриклассовое разнообразие, т.к. модель обучается на вариативности:

- различные расстояния до камеры (масштаб)
- небольшие повороты кисти (yaw/pitch/roll)
- различное освещение и фон
- разные темпы движения (если сборка включает динамику между кадрами)
- разные исполнители (если есть доступ)

### 5.3. Контроль качества и отбраковка

Для обучения допускаются только валидные примеры:

- длина вектора **строго 63**
- файл должен быть корректно читаем `numpy.load`
- желательно исключать кадры с частично потерянными landmarks (если это проявляется в аномальных значениях)

### 5.4. Получить один снимок кисти в формате `.npy`

Один пример датасета (вектор признаков) можно сохранить с веб-камеры скриптом `scripts/create_dataset.py`:

```bash
python scripts\create_dataset.py
```

Далее:

1. Введите `gesture_id` (номер класса жеста).
2. Появится окно `Hand Tracking`.
3. Когда MediaPipe уверенно найдет кисть, нажмите `Space` (клавиша с `keyCode == 32`).
4. Скрипт сохранит один файл `.npy` и выведет путь к нему в консоль.

Что лежит внутри `.npy`:

- это вектор признаков длины **63** (`(63,)`), полученный из `21 landmark × 3 координаты (x,y,z)`;
- перед сохранением координаты нормализуются относительно `wrist` (landmark `0`);
- данные сохраняются как `float`/`float32`.

## 6. Построить индекс и сделать stratified split

Построить индекс и сделать stratified split:

```bash
python scripts/split_dataset.py --test-size 0.2 --seed 42
```

Результаты:

- `data/index.csv` — список всех объектов и их меток
- `data/splits/train.csv`, `data/splits/test.csv` — разбиение с **stratified**-условием (сохранение пропорций классов)
